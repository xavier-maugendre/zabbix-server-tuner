Architecture proxy — Actif vs Passif
ProxyMode=0 Proxy Actif — recommandé prod
→Le proxy initie la connexion vers le serveur (port 10051). Une seule règle firewall sortante depuis le site distant.
→Chaque proxy actif consomme 2 trappeurs simultanément sur le backend : 1 pour les données, 1 pour la sync config.
→Côté serveur : StartTrappers doit être ≥ (nb_proxies × 2) + agents actifs directs.
→Envoi données : DataSenderFrequency=1s. Sync config : ProxyConfigFrequency=10s. Le proxy contrôle les intervalles.
✓ Bonne pratique prod — firewall sortant uniquement
ProxyMode=1 Proxy Passif
→Le serveur initie la connexion vers le proxy. Nécessite que le port 10051 soit ouvert en entrée sur le proxy.
→Côté serveur : StartProxyPollers = nb_proxies × 2 (1 poller données + 1 poller config par proxy).
→Le serveur contrôle les intervalles via ProxyDataFrequency et ProxyConfigFrequency.
→Utile sur réseau interne maîtrisé (même DC, VPN managé). Moins adapté aux sites distants.
△ Réseau interne uniquement — éviter sur site distant
